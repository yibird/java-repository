Java 和 Golang 是一门基于 GC 的语言,手动内存管理容易导致程序错误,如指针错误、重复释放内存、内存泄漏等。通过 GC 自动管理内存分配和回收,可以减少手动管理内存的负担,降低了内存泄漏和内存溢出的风险。由于内存是有限的,当 JVM 尝试分配对象时,如果没有足够的内存可用,会触发 GC 尝试回收垃圾(内存)。垃圾是指在程序运行过程中不再使用的对象或资源,这些对象占用内存但不再被任何变量引用,导致其内存资源无法被再次利用。

## 1.Java 四种引用

在 Java 中,为了更好地控制对象的生命周期和垃圾回收行为,引入了强引用(Strong Reference)、软引用(Soft Reference)、弱引用(Weak Reference)、虚引用(Phantom Reference)四种不同强度的引用类型。每种引用类型在决定对象是否可达以及何时被垃圾回收器回收方面具有不同的规则。

### 1.1 强引用

强引用是最常见的引用类型,例如平时在代码中直接创建的对象引用就是强引用。只要强引用存在,垃圾回收器就不会回收被引用的对象,即使在内存不足的情况下也不会回收。这是造成 Java 中内存泄漏最常见的原因之一。

```java
String str = new String("Hello World"); // str 是一个强引用
```

### 1.2 软引用

软引用描述了有用但非必需的对象。软引用关联的对象,在内存足够时不会被回收,但在内存不足时,垃圾回收器会回收软引用指向的对象,以避免内存溢出。软引用通常用于实现内存敏感的缓存。

## 2.常见 GC 算法

### 2.1 垃圾判别算法

垃圾回收(Garbage Collection, GC)是自动管理内存的过程,旨在回收不再使用的对象所占用的内存空间。为了确定哪些对象是垃圾,即不再被程序引用的对象。引用计数法和可达性分析算法是两种用于判断对象是否为垃圾(即不再被使用)的内存管理技术,它们是垃圾回收(Garbage Collection, GC)中识别垃圾的核心算法。

#### 2.1.1 引用计数法

引用计数法算法的核心:为每个对象分配一个引用计数器,每当有一个新的引用指向该对象时,计数器加 1;当引用被移除或失效时,计数器减 1。当一个对象的引用计数器变为 0 时,表明没有任何引用指向该对象,因此可以判定该对象为垃圾并进行回收。引用计数法虽然实现简单,执行效率高,回收过程可立即执行(低延迟),但存在如下缺点:

- 需要额外的开销维护引用计数器:使用引用计数法会为每个对象分配一个引用计数器,这样会导致额外的存储空间。
- 无法处理循环引用的问题:即对象 A 引用对象 B,对象 B 也引用对象 A,即使这两个对象已经不再被其他对象引用,但由于相互引用,它们的引用计数都不为 0,因此不会被回收。由于引用计数法无法处理循环引用,导致 Java 垃圾回收器并未使用这种算法判别垃圾。

#### 2.1.2 可行性分析算法

可达性算法基于图论中的可达性概念,将对象间的引用关系看作图中的边,从一组称为 GC Roots(垃圾收集根)的对象开始,沿着这些引用关系遍历,能够到达的对象被认为是可达的,反之则是不可达的,可被回收。
