## 1.事务基础

### 1.1 本地事务的定义

本地事务(Local Transaction)是指在单个数据库或单个资源管理器中执行的事务。它涉及对一个数据库的多个操作,这些操作被视为一个单一的工作单元,要么全部成功,要么全部失败,以确保数据的一致性和完整性。本地事务遵循 ACID 特性:

- 原子性(Atomicity):事务中的所有操作要么全部完成,要么全部不完成。事务一旦开始,就必须全部执行完成,如果在执行过程中出现任何错误,所有已执行的操作都必须回滚。
- 一致性(Consistency):事务开始前和结束后,数据库的状态必须是正确的。事务的执行不能破坏数据库的完整性约束。
- 隔离性(Isolation):一个事务的执行不能被其他事务干扰,不同事务之间相互独立。
- 持久性(Durability):事务一旦提交,其对数据库的修改就会永久保存下来,即使系统发生故障,数据也不会丢失。
